You are a Senior Full-Stack Engineer, SEO Engineer, and Marketplace Architect.

Project: https://www.codexstocks.org

Objective:
Implement a SOLD ACCOUNT LIFECYCLE system that improves user experience, SEO, and inventory management.

Current Business Logic:

* Every PUBG account is unique.
* Once sold, the account will never be sold again.
* We do NOT want sold accounts to remain permanently visible in search results.
* We also do NOT want visitors from Google to land on a dead page immediately.

Implementation Requirements:

PHASE 1 — ACTIVE
When an account is available:

* status = active
* Account page works normally.
* Account appears in sitemap.xml.
* Product schema availability = InStock.

PHASE 2 — SOLD
When an account is sold:

* status = sold
* Automatically remove the URL from sitemap.xml.
* Keep the page accessible for 14 days.
* Show a clear message:

"This account has been sold."

* Hide all purchase buttons.
* Show a "View Similar Accounts" section.
* Automatically display related active accounts based on:

  * Price range
  * Similar inventory
  * Similar account type
  * Similar popularity

SEO Requirements:

* Keep canonical tag intact.

* Keep Product schema.

* Change availability to:
  https://schema.org/OutOfStock

* Do not add noindex during the sold period.

* Preserve existing structured data.

PHASE 3 — ARCHIVED
After 14 days:

* Automatically archive the sold account.
* Remove it completely from public listings.
* Return HTTP 410 Gone.
* Remove all Product schema output.
* Display a custom 410 page:

"This account is no longer available."

* Include:

  * Button to browse active accounts
  * Link to marketplace homepage

SEO Requirements:

* HTTP status must be 410 (not 404).
* No redirects to homepage.
* No redirects to /accounts.
* No soft-404 behavior.

AUTOMATION

Create an automated process that:

* Detects sold accounts.
* Tracks sold date.
* Archives after 14 days.
* Serves HTTP 410 automatically.

SITEMAP RULES

Include only:

* status = active
* visibility = public

Exclude:

* sold
* archived
* private

DELIVERABLES

Provide:

1. Database changes
2. Files modified
3. API changes
4. Frontend changes
5. SEO changes
6. Structured data changes
7. Automation implementation
8. Testing results
9. Example user journey
10. Example Googlebot journey

Do not break existing SEO functionality, canonical tags, sitemap generation, prerendering, or structured data.
