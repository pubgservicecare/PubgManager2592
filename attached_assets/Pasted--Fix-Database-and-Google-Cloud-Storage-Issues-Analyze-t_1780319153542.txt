# Fix Database and Google Cloud Storage Issues

Analyze the existing codebase and implement the following fixes without breaking existing functionality.

## Goal 1: Prioritize External Neon Database Over Replit Managed PostgreSQL

### Problem

Replit automatically injects and locks the `DATABASE_URL` environment variable, making it impossible to override directly.

We want the application to prefer an external Neon PostgreSQL database whenever `NEON_DATABASE_URL` is available.

### Required Changes

#### File: `lib/db/src/index.ts`

Replace database connection logic with:

```ts
const connectionString =
  process.env.NEON_DATABASE_URL ||
  process.env.DATABASE_URL;
```

### Requirements

* If `NEON_DATABASE_URL` exists, always use it.
* Otherwise fall back to Replit's `DATABASE_URL`.
* Enable SSL automatically when using Neon.
* Preserve compatibility with the existing database client.

---

## Goal 2: Update Drizzle Configuration for Neon

### Problem

`lib/db/drizzle.config.ts` currently reads only `DATABASE_URL`.

As a result, commands such as:

```bash
pnpm db push
pnpm db generate
```

still target the Replit database instead of Neon.

### Required Changes

#### File: `lib/db/drizzle.config.ts`

Update connection selection:

```ts
const url =
  process.env.NEON_DATABASE_URL ||
  process.env.DATABASE_URL;
```

### Requirements

* Schema generation and migrations must target Neon whenever `NEON_DATABASE_URL` exists.
* Maintain backward compatibility.

---

## Goal 3: Fix Google Cloud Storage Upload and Retrieval Bugs

### Bug A: Empty Bucket Name

Current implementation sometimes generates paths like:

```ts
// incorrect
/${bucket}/uploads/${uuid}
```

which later become:

```txt
//bucket/uploads/uuid
```

causing:

```txt
A bucket name is needed to use Cloud Storage
```

### Fix

Remove the leading slash.

Use:

```ts
const fullPath = folderPath
  ? `${bucket}/${folderPath}/${uuid}`
  : `${bucket}/uploads/${uuid}`;
```

Ensure bucket parsing never produces an empty bucket name.

---

### Bug B: Signed URL Stored in Database

Current behavior stores the complete signed URL:

```txt
https://storage.googleapis.com/...
```

These URLs expire after ~15 minutes.

### Fix

Store a permanent object path instead:

```ts
return `/objects/${bucketRelativePath}`;
```

Example:

```txt
/objects/uploads/abc123
```

Never store signed URLs in the database.

Signed URLs should only be used for temporary upload access.

---

### Bug C: Retrieval Fails Without Folder Path

Current retrieval logic assumes `gcsFolderPath` always exists and throws:

```ts
GcsNotConfiguredError
```

when it is empty.

### Fix

Support both cases:

```ts
const objectName = folderPath
  ? `${folderPath}/${entityId}`
  : entityId;
```

Folder path should be optional.

---

## Expected Final Flow

```txt
Browser
   ↓
Upload Request
   ↓
API generates signed GCS upload URL (temporary)
   ↓
Browser uploads directly to GCS
   ↓
Database stores:
   /objects/uploads/{uuid}
   (permanent path)
   ↓
Image Request
   ↓
/api/storage/objects/uploads/{uuid}
   ↓
Backend resolves GCS object
   ↓
Streams image from bucket
```
